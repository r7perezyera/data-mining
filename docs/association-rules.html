El problema Market-Basket

  • Provista una base de datos de transacciones, encontrar reglas que predicirn la ocurrencia de un elemento basado en la ocurrencia de otros elementos en la transaccin.
  • La minera de conjuntos de elementos (itemsets) y reglas de asociacin es un enfoque popular con amplia investigacin para descubrir relaciones interesantes entre variables en bases de datos grandes.

Reglas de asociacin

  • Mtodo de aprendizaje de mquina para descubrir relaciones interesantes entre variables en bases de datos grandes. Identifica reglas empleando mtricas de inters.
  • Desarrollado por Petr Hjek.
  • Popularizado por Rakesh Agrawal, Tomasz Imieliski y Arun Swam durante los 90s por sus publicaciones relacionadas con el descubrimiento de tendencias en comn entre productos en los registros de sistemas de punto de venta en supermercados.
  • El orden de los elementos no es relevante para reglas de asociacin.

  • Colocacin de productos.

  • Anuncios.

  • Ventas.

Principales algoritmos de reglas de asociacin

Algoritmo Descripcin
Apriori Emplea una estratrategia de amplitud primero para contar el soporte en itemsets y emplea una funcin de generacin de candidatos que explota la propiedad de cierre del soporte de sus descendientes.
Eclat Equivalence Class Transformation, es un algoritmo de profundidad primero basado en interseccin de conjuntos.

Concepto y elementos de una regla de asociacin

Regla de asociacin: X => Y donde X, Y pertenecen a I.

I es un conjunto de elementos binarios, llamado items. D es un conjunto de transacciones, denominado base de datos.

Cada regla consta de dos conjuntos de elementos, conocidos como itemsets, X y Y.

X es el antecedente o lado izquierdo (left-hand-side, LHS). Y es el consecuente o lado derecho (right-hand-side, RHS).

Una regla requiere del soporte de una elevanda cantidad de transacciones para considerarse relevante estadsticamente.

Ejemplos de reglas de asociacin

{milk} => {bread}

{milk, bread} => {butter}

Nota: Las reglas de asociacin cuentan con mtricas de support, confidence, entre otras.

Nota: La implicacin significa co-ocurrencia, no causalidad.

Observaciones del ejemplo

  • Todas las reglas son particiones binarias del mismo itemset {milk, beer, etc}
  • Las reglas que se original del mismo itemset tienen el mismo soporte con distinto nivel de confianza.
  • Es posible desacoplar los requerimientos de support y confidence.

Encontrar Reglas de Asociacin

Enfoque de dos pasos:

  1. Generacin de itemsets frecuentes.
  • Generar todos los itemsets donde support >= minsup.
  1. Generacin de reglas.
  • Generar reglas de alta confianza para cada itemset frecuente, donde cada regla sea una particin binaria de un itemset frecuente.

La generacin de itemsets es cara desde la perspectiva computacional!

item

Un conjunto de atributos binarios I = {i2, i2, …, in}

Para el caso de nuestro ejemplo: I = {milk, bread, butter, beer}

database

Conjunto de transacciones D = {t1, t2, …, tm}

Cada transaccin en D tiene un ID nico de transaccin y contiene un subconjunto de los elementos en I.

rule

Implicacin de la forma X => Y donde X, Y <= I y X interseccin Y = [conjunto vaco]

El conjunto de items (referidos como itemsets) X y Y se denominan antecedente (left-hand-side o LHs) y consecuente (right-hand-side o RHS) de la regla.

Frequent Itemset

  • itemset
  • Coleccin de uno o ms items
    • Ejemplo: a, b, c
  • support Proporcin de transacciones que contienen un itemset
  • itemset frecuente
  • Un itemset cuyo support sea mayor o igual a un umbral minsup

Association Rule

  • Association rule.
  • Una expresin de implicacin de la forma X -> Y, donde X y Y son itemsets.
  • Ejemplo: {LHS} -> {RHS}
  • Mtricas de evaluacin de reglas:
  • support.
  • confidence.

Mtricas de las reglas de asociacin

Mtrica Representacin Descripcin
Support supp(X) Proporcin de transacciones en el conjunto de datos que contienen el itemset.
Confidence conf(X=>Y) Estimacin P(Y
Confidence cont. la probabilidad de encontrar el RHS de la regla en transacciones bajo la condicin que stas tambin incluyan LHS
Lift lift(X => Y) Desviacin de support para la regla completa a partir del soporte esperado bajo independencia provisto los soportes de LHS y RHS.

Ejemplo de referencia

ID de transaccin items
1 milk, bread
2 bread, butter
3 beer
4 milk, bread, butter
5 bread, butter

Conjunto de items = I = {milk, bread, butter, beer} Nmero de transacciones = 5

Support

Proporcin de transacciones en el conjunto de datos que contienen el itemset.

p.e. considerando el ejemplo de referencia para el itemset X = {milk, bread} su support supp(X) es 2/5 = 0.4 debido a que el itemset {milk, bread} ocurre en el 40% de las transacciones.

Confidence

Se interpreta como la probabilidad de encontrar el RHS (Y) de la regla en transacciones bajo la condicin que stas tambin incluyan LHS (X).

Formalmente, conf(X => Y) = supp(X unin Y) / supp(X)

p.e. considerando el ejemplo de referencia para la regla {milk, bread} => {butter} tiene una confianza de 0.5 es decir, en 50% de las transacciones que contienen milk y bread la regla es correcta (incluyen butter).

Requisitos de las reglas de asociacin

Las reglas de asociacin requieren que se cumpla con valores mnimos de support y confidence al mismo tiempo.

Los algoritmos de reglas de asociacin requieren que el dataset cuente con atributos categricos.

Se recomienda crear variables derivadas de tipo categrico cuando el dataset tenga atributos numricos.

Paquete arules (lenguaje R)

El paquete arules provee de infraestructura bsica para crear y transformar datasets, as como para analizar los itemsets y reglas resultantes. arules tambin incluye interfaces a dos algorimos de minera rpida, implementaciones en lenguaje C de Apriori y Eclat por Christian Borgelt. Estos algoritmos pueden ser empleados para minar itemsets frecuentes, itemsets de frecuencia mxima, itemsets cerrados frecuentes y reglas de asociacin.

  • arules emplea representaciones de matrices dispersas para minimizar el consumo de memoria.

  • Diseado considerando extensibilidad tanto en interfaces con nuevos algoritmos y en la adicin de nuevos tipos de mtricas y asociaciones de inters.

Mtodos principales en arules

El resultado de minar datos de transacciones en a rules son asociaciones.

Conceptualmente, las asociaciones son un conjunto de objetos que describen las relaciones entre un conjunto de items (p.e. un itemset o una regla) que tienen asignados valores para distintas mtricas de calidad (p.e. confidence y lift), o otras mtricas (p.e. ingresos cubiertos por la asociacin).

Funcionalidad comn en arules

  • summary() provee de una panorama general de conjunto de datos e inspect() despliegan asociaciones individuales,
  • length() para obtener el nmero de elementos de un set,
  • items() para obtener para cada asociacin un conjunto de elementos items involucrados en la asociacin (p.e. la unin de items en el LHS y RHS por cada regla),
  • sort() para ordenar el conjunto de datos set empleando valores de distintas mtricas de calidad,
  • extraccin de subconjuntos por medio de [] y (),
  • operacin de conjuntos (union(), intersect(), setequal()), y
  • encontrar los elementos que correspondan a dos conjuntos (match()).
